This guide walks you through creating a fully functional search component using the **local data** mode — the simplest way to get started. You pass a JavaScript array directly to the constructor and buscadorDinamico handles rendering, filtering, and pagination.

## Build your first search component

<Steps>
  <Step title="Create the HTML container">
    Add a single wrapper `<div>` with the `.app-search` class anywhere in your page body:

    ```html
    <div class="app-search"></div>
    ```

    You do not need to add any child elements — the library generates the input field, results list, and pagination footer automatically.
  </Step>

  <Step title="Include the script and stylesheet">
    In your `<head>`, link the stylesheet:

    ```html
    <link rel="stylesheet" href="src/css/app.css">
    ```

    At the bottom of `<body>`, include the script:

    ```html
    <script src="src/js/app.js"></script>
    ```
  </Step>

  <Step title="Initialize the search class">
    After the script tag, instantiate the `search` class with your data:

    ```javascript
    const mySearch = new search({
      element: ".app-search",
      itemsPerPage: 5,
      data: [
        { name: "Venezuela",  descripcion: "The richest country in oil reserves." },
        { name: "Colombia",   descripcion: "Rich in cultural heritage and natural landscapes." },
        { name: "Brazil",     descripcion: "The largest country in South America." },
        { name: "Argentina",  descripcion: "Known for its passion for football and culture." },
        { name: "Chile",      descripcion: "A country with a wide variety of natural landscapes." },
        { name: "Peru",       descripcion: "Rich in cultural heritage and natural diversity." },
        { name: "Ecuador",    descripcion: "Natural variety and rich cultural traditions." },
        { name: "Uruguay",    descripcion: "Known for football culture and quality of life." }
      ],
      renderizarItem: function(element) {
        return `
          <div class="info-item">
            <h5>${element.name}</h5>
            <p>${element.descripcion}</p>
          </div>
        `;
      }
    });
    ```

    The constructor signature in full:

    ```javascript
    new search({
      element,        // {string}   CSS selector for the wrapper element
      itemsPerPage,   // {number}   Results per page (default: 10)
      data,           // {Array}    Local data array — objects with any shape
      renderizarItem  // {Function} Receives one item object, returns an HTML string
    })
    ```

    <Tip>
      The search input uses a **500 ms debounce** — filtering only runs after the user pauses typing, which keeps the DOM updates smooth. Search results are also **cached for 5 minutes**: if you type the same term again, buscadorDinamico returns the stored result instantly without re-filtering the data. The cache key includes the search term, the current page number, and the data source type.
    </Tip>
  </Step>

  <Step title="See what gets rendered">
    After `new search(...)` runs, the library injects three child elements into your `.app-search` container:

    - **Input area** (`.input-search`) — a `<search>` element containing a `<label>` and an `<input type="text">` with the placeholder `Ingrese palabra clave...`.
    - **Results list** (`.items-search`) — a scrollable `<main>` element. Each matched result becomes a `<section class="items">` whose inner HTML is produced by your `renderizarItem` function.
    - **Pagination footer** (`.index-search`) — a `<footer>` containing a `<ul class="pagination">`. Pagination buttons appear for **first**, **previous**, **current**, **next**, and **last** pages. The current page button is highlighted with the `.page-selected` class.
  </Step>
</Steps>

```html
<!DOCTYPE html>

        { name: "Peru",       descripcion: "Rich in cultural heritage and natural diversity." },
        { name: "Ecuador",    descripcion: "Natural variety and rich cultural traditions." },
```

## Choose your data mode

buscadorDinamico supports three ways to provide data. Swap the constructor configuration to switch modes.

<Tabs>
  <Tab title="Local data">
    Pass a JavaScript array to `data`. The library filters it client-side on every keystroke (debounced) and paginates the results.

    ```javascript
    const mySearch = new search({
      element: ".app-search",
      itemsPerPage: 10,
      data: [
        { name: "Item 1", descripcion: "Description 1" },
        { name: "Item 2", descripcion: "Description 2" }
      ],
      renderizarItem: function(element) {
        return `<p><strong>${element.name}</strong> — ${element.descripcion}</p>`;
      }
    });
    ```

    Filtering checks every property value of each object, so items match if the search term appears anywhere in any field.

    For a full walkthrough, see the [Local Data guide](/guides/local-data).
  </Tab>

  <Tab title="DOM indexing">
    Omit `data` entirely. The library reads the `data-*` attributes from existing `<section class="items">` elements already in the DOM.

    ```html
    <div class="app-search">
      <main class="items-search scroll-personalize">
        <section class="items"
          data-name="Venezuela"
          data-descripcion="The richest country in oil reserves.">
        </section>
        <section class="items"
          data-name="Colombia"
          data-descripcion="Rich in cultural heritage.">
        </section>
      </main>
    </div>
    ```

    ```javascript
    const mySearch = new search({
      element: ".app-search",
      itemsPerPage: 10,
      renderizarItem: function(element) {
        return `<p><strong>${element.name}</strong> — ${element.descripcion}</p>`;
      }
    });
    ```

    Each `data-*` attribute becomes a property on the item object, so `data-name` is available as `element.name` inside `renderizarItem`.

    For a full walkthrough, see the [DOM Indexing guide](/guides/dom-indexing).
  </Tab>

  <Tab title="AJAX">
    Set `procesServer: true` and pass an object to `data` describing the request. The library fetches results from your server on initialization and on every search term or page change.

    ```javascript
    const mySearch = new search({
      element: ".app-search",
      procesServer: true,
      itemsPerPage: 10,
      data: {
        url: "./api/search.php",
        method: "POST",
        body: {
          page: 1,
          searchTerm: ""
        }
      },
      renderizarItem: function(element) {
        return `<p><strong>${element.name}</strong> — ${element.descripcion}</p>`;
      }
    });
    ```

    Your server must return JSON in this shape:

    ```json
    {
      "data": [],
      "page": 1,
      "countPage": 10
    }
    ```

    The `itemsPerPage` value is automatically appended to the request body — any `itemsPerPage` key you set in `body` is overwritten by the class instance value.

    For a full walkthrough, see the [AJAX guide](/guides/ajax-mode).
  </Tab>
</Tabs>
